# 週間タイムトラッカー

Google カレンダーと連携した週間スケジュール管理ツールです。

---

## 主な機能

### 週間カレンダービュー
- 月〜日の7日間を時間軸（7時〜22時）で表示
- 当日の日付をハイライト表示
- `‹` / `›` ボタンで前週・翌週に移動可能
- カレンダーのセルをクリックすると、その時間帯でイベント追加フォームが開く

### Google カレンダー同期
- Claude に依頼することで GCal の最新データを HTML に反映
- 同期日時をヘッダーに表示
- GCal から取得したイベントには `GCal` バッジを表示

### イベント管理
- **追加**：「ローカルにイベントを追加」ボタンからタイトル・日時・カテゴリ・メモを入力して保存
- **削除**：イベントをクリックまたは `✕` ボタンで削除
- ローカル追加イベントは `localStorage` に保存され、ページをリロードしても保持される
- ローカル追加後は Claude への依頼文が自動生成されコピー可能（GCal への反映に使用）

### カテゴリ
| カテゴリ | 色 |
|---|---|
| 会議・面談 | 青 |
| 集中作業 | 緑 |
| 管理業務 | 黄 |
| 個人用 | ピンク |
| 学習 | 紫 |
| 休憩 | グレー |

### 統計サマリー
画面上部に今週のサマリーをカード表示：
- **今週の合計**：全イベントの合計時間と件数
- **会議・面談**：会議カテゴリの合計時間と割合
- **集中作業**：集中作業カテゴリの合計時間と割合
- **空き時間**：週40時間想定での残り時間

### カテゴリ別グラフ
- ドーナツ型の円グラフでカテゴリ比率を可視化
- 各カテゴリの時間・割合をバー付きで一覧表示

### 週間レビューチェックリスト
週の振り返り用チェック項目：
1. 今週の会議を記録した
2. 集中作業の時間を振り返った
3. 時間の無駄を1つ特定した
4. 来週の予定をGCalに入れた
5. 休憩・回復時間を確保した

チェック状態は `localStorage` に保存され、次回以降も引き継がれます。

## Slack 連携（Claude MCP 経由）

Claude の MCP（Model Context Protocol）を通じて、スケジュール情報を Slack に通知できます。コードの変更は不要で、Claude に自然言語で依頼するだけで実行されます。

### できること

| 操作 | 依頼例 |
|---|---|
| 今週の予定を通知 | 「今週のスケジュールを #general に送って」 |
| 特定イベントを共有 | 「明日の面談をチームに Slack で知らせて」 |
| 週次サマリーを投稿 | 「今週の作業時間サマリーを Slack に投稿して」 |
| メッセージを下書き | 「来週の予定を Slack の下書きに保存して」 |
| スケジュール送信 | 「月曜朝9時に今週の予定を #times-fujiyama に送るよう予約して」 |

### 運用フロー

```
1. weekly-tracker-live.html でスケジュールを確認
        ↓
2. Claude に Slack 通知を依頼（自然言語でOK）
        ↓
3. Claude が MCP 経由で Slack API を呼び出し
        ↓
4. 指定チャンネル／DMにメッセージが投稿される
```

### 依頼の具体例

**今週のサマリーを投稿**
```
今週のスケジュールを見て、作業時間のサマリーを #times-fujiyama に投稿して
```

**会議前のリマインダーを送信**
```
今日14時の面談を @yamada に Slack で共有して
```

**週次レポートを予約送信**
```
毎週月曜9時に今週の予定を #team-schedule に送るよう設定して
```

### 注意事項

- Claude MCP の Slack 連携が有効になっている必要があります
- 投稿先チャンネルへの権限が必要です
- スケジュール送信（予約投稿）は Slack の予約メッセージ機能を使用します

---

## 使い方

1. `weekly-tracker-live.html` をブラウザで開く
2. GCal の最新データを反映したい場合は Claude に「HTMLを更新して」と依頼
3. ローカルでイベントを追加したい場合は「ローカルにイベントを追加」ボタンを使用
4. 追加後に表示される依頼文をコピーして Claude に送ると GCal にも反映される

---

## 技術構成

- 純粋な HTML / CSS / JavaScript（フレームワーク不使用）
- データ保存：`localStorage`（ローカル追加イベント・チェック状態）
- フォント：Noto Sans JP、DM Mono（Google Fonts）
- グラフ：Canvas API
